## What got fixed (see commits on `fix/u1-test-suite-triage`)

1. **Flat-shim star-import re-export gap** (dominant cluster, ~250 failures) — 12 canonical `cli/<pkg>/<name>.py` modules got an explicit `__all__` so their one-line `cli/<name>.py` shims (`from ...cli.<pkg>.<name> import *`) stop silently dropping every underscore-prefixed symbol. 23 flat shims were missing `if __name__ == "__main__": main()` guards, breaking `python -m backlink_publisher.cli.<name>` (webui's `run_pipe()` dispatch path). ~20 tests were patching/monkeypatching the flat shim's copy of a name instead of the canonical module where the real call site resolves it (silent no-op mock) — repointed.
2. **Real source bugs** (not shim artifacts): `cli/publish/report_anchors.py` importing a renamed-away `cli.report_engine` module; `cli/publish/verify_dofollow.py`'s built-in-catalog path arithmetic one directory level too shallow after the reorg; `_util/paths.py::_resolve_config_dir()` re-importing from its own defining module instead of `backlink_publisher.config` (defeating the documented test-monkeypatch contract); `webui_app/routes/llm.py` missing the `_safe_post_json` import its own docstring promises; `webui_app/helpers/url_meta.py::detect_platform()` returning `'medium'` from every branch including the one a comment says was flipped to `'blogger'`; a dead `pyproject.toml` `footprint` entry point.
3. **Stale tests after the SPA migration** (`782833e1`) — ~9 files, ~140+ test cases repointed from the now-redirecting bare route to its `/jinja` fallback, or converted to assert the 302 contract. Also restored route-coverage-gate compliance (`test_webui_core_routes.py::test_every_route_has_at_least_one_contract_test`) by adding one lightweight redirect-contract test per migrated route.
4. **Guard/allowlist drift**: CSRF-mutation grandfather list, CSRF-only-route snapshot count, raw-requests allowlist line-number drift, raw-home-path-primitive allowed-module pointer (moved from `config/loader.py` to `_util/paths.py` per P14 A1-A3 but the test guard's pointer never followed).
5. **Test bit-rot**: `test_webui_health_publish_metrics.py` hardcoded a fixed calendar date as "now" for events feeding a rolling 30-day window — fixed to `datetime.now(UTC)`.
6. New guard: `tests/test_cli_shim_reexports.py` (verified red→green) to catch future `__all__` regressions on the 12 fixed modules.

## Residual classification

### A. Windows-local-only artifacts (majority of residue — NOT expected to fail on CI's `ubuntu-latest` runner)

Confirmed via `.github/workflows/ci.yml`: all CI jobs run on `ubuntu-latest`. These clusters were each individually traced to a Windows-specific root cause, not a source bug:

- **NTFS chmod semantics** (~40 tests): `os.chmod(path, 0o600)` is a no-op on NTFS; files report `0o666`/dirs `0o777` regardless of the requested mode. Files: `test_secrets.py`, `test_image_gen_token_rotation.py`, `test_save_config_section_taxonomy_canary.py`, `test_canary_store.py`, `test_comment_outreach_status_store.py`, `test_credential_service.py`, `test_registry_credential_saver.py`, `test_cli_health_check.py`, `test_config_llm_sidecar.py`, `test_dedup_digest.py`, `test_dedup_connection.py`, `test_reliability_circuit.py`, `test_webui_llm_test_persist.py`, `test_provider.py`, `test_settings_service.py`, `test_purge_removed_credentials.py`, and others.
- **`_compat/fcntl.py` EACCES→EAGAIN translation gap** (~15 tests): `msvcrt.locking()`'s `PermissionError` doesn't carry `winerror == 33` in every contention path this compat shim checks, so lock-contention crashes instead of signalling "held". Real `fcntl` on Linux doesn't go through this shim at all. Files: `test_dedup_operator_verbs.py`, `test_dedup_adjudicate.py`, `test_dedup_force_manifest.py`, `test_reliability_circuit_crossproc.py`, `test_io_atomic_crossproc.py`, `test_idempotency_backfill.py`, `test_frw_login.py`.
- **Path-separator string comparisons** (~10 tests): tests compare `str(Path(...))` against POSIX literals (`/tmp/x`), or build relpath sets from `Path.relative_to()` and compare against forward-slash-keyed allowlists/frozensets — both produce backslash strings on Windows. Files: `test_fail_closed_resolver.py`, `test_no_raw_home_path_primitives.py` (2 of its 5 failures — the other 3 were the real `_RAW_HOME_ALLOWED_MODULE` + `config/loader.py:178` gap, now fixed), `test_no_inner_import_shadowing.py`, `test_cli_exit_code_literals.py`, `test_config_managed_root_subsection_roundtrip.py`, `test_config_echo.py` (partially), `test_no_raw_requests_outside_http_client.py` (line-number drift already fixed; residual is pure path-separator).
- **Windows tempfile/handle quirks** (~3 tests): sqlite handle still open when `TemporaryDirectory` cleans up (Windows can't delete open-handle files; Linux can); symlink creation needs dev-mode/admin privilege on Windows.
- **Timestamp-collision snapshot rotation** (flaky, 1-4 tests depending on run): filenames derived from `strftime(...%f)` collide when consecutive calls land in the same tick of Windows's coarser clock resolution, so a loop meant to produce N distinct snapshot files silently overwrites down to 1. Reproduced directly: `time.monotonic() > time.monotonic()` computed microseconds apart returns `False` on this machine. Linux's finer-grained monotonic clock does not hit this. Files: `test_config_safety_net.py`, `test_safe_write_substrate.py::test_rotate_snapshots`, `test_image_gen_token_rotation.py` (two-archives cases), `test_cli_recheck_backlinks.py::test_batch_budget_exhaustion_defers_remaining` (same root clock-granularity cause, different symptom — a `deadline = time.monotonic() + 0.0` check never trips within the same batch).
- **10 ERRORs**: `test_phase0_seal_hook.py` fixture setup shells out to `scripts/install-pre-push-hook.sh` via `bash`, which exits 127 (command-not-found) under this Windows Git-Bash/PATH interop — pre-existing, unrelated to U1.

### B. Real, non-Windows-specific bugs found — diagnosed but NOT fixed this pass (flagged for dedicated follow-up)

- **`backlink_publisher/__init__.py` facade shadowing** (`test_public_facade_resolvable.py`, 2 tests): `bp.dispatch`/`bp.validate` are meant to resolve lazily via module `__getattr__` (PEP 562), but real subpackages `src/backlink_publisher/dispatch/` and `src/backlink_publisher/validate/` exist with the same names. The first time *any* code in the process does `from backlink_publisher.validate import ...` (e.g. `cli/validate_backlinks.py`, `sdk/api.py`), Python's import machinery permanently binds `backlink_publisher.validate` to the submodule on the package object, and `__getattr__` is never consulted again for that name — order-dependent, reproduces under `pytest -k` across the full suite but not when running the single test file alone. Fixing this needs an architectural decision (rename either the facade attributes or the subpackages) that's out of scope for a triage pass; not attempted here to avoid a risky, wide-blast-radius rename under time pressure.
- **`test_benchmarks.py::test_benchmark_publish_50_rows_dry_run`**: the benchmark's synthetic payload (`link_count: 1`) fails current `publish-backlinks` validation, which requires `link_count` between 6 and 8, plus a missing required `seo` output field — the fixture payload has drifted out of sync with the current schema. Needs someone familiar with the current payload schema to regenerate the fixture; not attempted here since a wrong fix could mask a real validation regression.
